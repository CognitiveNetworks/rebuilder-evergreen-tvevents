# Feature Parity Matrix

> Complete list of every user-facing feature, integration, and workflow in the legacy application.
> Each feature gets a status. The rebuild is not complete until every **Must Rebuild** and **Rebuild Improved** feature passes acceptance testing.

## Features

| Feature | Legacy Behavior | Status | Target Behavior | Acceptance Criteria | Notes |
|---|---|---|---|---|---|
| POST `/` event ingestion | Receives TV event JSON payloads via POST to `/`, processes event through validation → transformation → blacklist check → obfuscation → Firehose delivery pipeline | **Rebuild Improved** | POST `/v1/events` with same query params (`tvid`, `event_type`) and JSON body. FastAPI with Pydantic request models. Same validation → transformation → obfuscation pipeline; Kafka delivery replaces Firehose | POST to `/v1/events` accepts valid payloads and returns typed Pydantic response with event ID; invalid payloads return 422 with structured error | Phase 1 — primary endpoint; backward-compatible payload format |
| GET `/status` health check | Returns plain text "OK" with 200 status regardless of dependency state | **Rebuild Improved** | GET `/health` checks Kafka broker, RDS, and Redis connectivity. Returns 200 when healthy, 503 with structured JSON when any critical dependency is unreachable | `/health` returns 200 when all dependencies up; returns 503 when Kafka, RDS, or Redis is down; response body includes per-dependency status | Phase 1 |
| HMAC security hash validation | `cnlib.token_hash.security_hash_match(tvid, h, salt)` validates hash sent by TV against `T1_SALT` env var. Uses `==` string comparison | **Rebuild Improved** | Standalone `domain/validation.py security_hash_match()` using `hmac.compare_digest()` for constant-time comparison. Salt loaded from AWS Secrets Manager (injected via `T1_SALT` env var) | Valid HMAC accepted; invalid HMAC returns 400 with `TvEventsSecurityValidationError`; timing-safe comparison verified (no timing side-channel) | Phase 1 — security improvement over `==` comparison |
| Required parameter validation | Checks `TvEvent` envelope contains `tvid`, `client`, `h`, `EventType`, `timestamp`; returns 400 if any missing | **Rebuild Improved** | Pydantic model with required fields enforces presence at deserialization. Missing fields return 422 with field-level error details | Missing any required field returns 422 with structured error identifying the missing field(s) | Phase 1 — Pydantic replaces manual checks |
| Parameter match check | Verifies `tvid` and `event_type` from URL query params match values in JSON body `TvEvent` | **Must Rebuild** | Same cross-validation between URL params and body. Implemented in `domain/validation.py` | Mismatched `tvid` or `event_type` between URL and body returns 400; matched params pass validation | Phase 1 |
| Timestamp validation | Checks `timestamp` is valid datetime; handles both millisecond and second epoch formats | **Must Rebuild** | Same dual-format timestamp validation in `domain/validation.py` | Millisecond timestamps accepted; second timestamps accepted; invalid timestamps (non-numeric, negative, far-future) rejected with 400 | Phase 1 |
| ACR_TUNER_DATA event type | Validates `channelData` (`majorId`, `minorId`), `resolution` (`vRes`, `hRes`), `programData`; handles Heartbeat events with no channel data | **Must Rebuild** | Same validation logic in `domain/event_types.py AcrTunerDataEventType`. Pydantic models for channelData, resolution, programData | ACR payloads with valid channelData/resolution/programData accepted; Heartbeat events (no channelData) accepted; invalid payloads rejected with 422 | Phase 1 |
| PLATFORM_TELEMETRY event type | Validates `PanelData` with JSON Schema: `Timestamp` present, `PanelState` is ON or OFF, `WakeupReason` is 0–128 | **Rebuild Improved** | Same validation rules via Pydantic model in `domain/event_types.py PlatformTelemetryEventType`. Replaces `jsonschema` library with Pydantic | PanelData with valid Timestamp/PanelState/WakeupReason accepted; out-of-range WakeupReason (>128) rejected; missing Timestamp rejected | Phase 1 — Pydantic replaces jsonschema |
| NATIVEAPP_TELEMETRY event type | Validates `Timestamp` in `EventData`, extracts `Namespace` and `AppId` for output | **Must Rebuild** | Same validation and extraction in `domain/event_types.py NativeAppTelemetryEventType` | EventData with valid Timestamp accepted; Namespace and AppId extracted into output JSON; missing Timestamp rejected | Phase 1 |
| Event type dispatch | Maps `EventType` string (e.g., `"ACR_TUNER_DATA"`) to `EventType` class via `event_type_map` dictionary | **Must Rebuild** | Same dispatch pattern in `domain/event_types.py` mapping string → class | All three event types dispatched correctly; unknown EventType returns 400 | Phase 1 |
| JSON flattening | Recursive flattening of nested `EventData` with key prefixes (e.g., `channeldata_majorid`, `paneldata_panelstate`) | **Must Rebuild** | Same recursive flattening logic in domain layer. Output key format preserved exactly for downstream compatibility | Flattened keys match legacy format byte-for-byte; nested structures 2+ levels deep flatten correctly with underscore-joined prefixes | Phase 1 — golden-file tests validate output |
| Output JSON generation | Flattens `TvEvent` fields (excluding `h`, `timestamp`, `EventType`), adds `tvevent_timestamp`, `tvevent_eventtype`, `zoo` env var, merges flattened `EventData` | **Must Rebuild** | Same output generation logic. Identical field names and structure for downstream consumer compatibility | Output JSON matches legacy format for all event types; `h` excluded, `timestamp` → `tvevent_timestamp`, `EventType` → `tvevent_eventtype`, `zoo` added from environment | Phase 1 — golden-file tests validate exact format |
| Blacklisted channel lookup | Queries RDS PostgreSQL `tvevents_blacklisted_station_channel_map` table for `channel_id` via `SELECT DISTINCT channel_id` | **Must Rebuild** | Same RDS query via `asyncpg` in `services/rds_client.py`. Read-only async connection pool | Query returns same `channel_id` set; connection pool reuses connections; query timeout configured | Phase 1 — same RDS, same table, same query |
| Blacklist cache (file-based) | Reads/writes `/tmp/.blacklisted_channel_ids_cache` as JSON file; fragile — lost on pod restart, not shared across pods | **Rebuild Improved** | Redis-backed cache via `rebuilder-redis-module` with TTL. Shared across all pods. Graceful degradation to RDS on cache miss | Cache hit returns blacklist set without RDS query; cache miss falls back to RDS; cache TTL expires and refreshes; cache shared across pods | Phase 1 — Redis replaces file cache per PRD |
| Blacklist cache (in-memory) | `_blacklisted_channel_ids` instance variable on the app class | **Rebuild Improved** | Redis cache serves as shared cache; in-memory set populated from Redis on startup and refreshed on TTL | In-memory lookup fast-path matches Redis state; refresh cycle updates in-memory set | Phase 1 |
| Blacklist cache initialization | Fetches full blacklist from RDS at startup; fails pod if RDS unreachable | **Must Rebuild** | Fetch from RDS → populate Redis cache at startup. Fail startup if RDS unreachable (same behavior) | App fails to start if RDS is unreachable during initialization; successful start populates Redis cache | Phase 1 — startup dependency preserved |
| Channel obfuscation | If `iscontentblocked` is true or `channelid` is in blacklist set, replaces `channelid`, `programid`, `channelname` with string `"OBFUSCATED"` | **Must Rebuild** | Same obfuscation logic in `domain/obfuscation.py`. Same fields, same `"OBFUSCATED"` replacement string | Blacklisted channels obfuscated in output; `iscontentblocked=true` obfuscated; non-blacklisted channels pass through unchanged; obfuscation applies to `channelid`, `programid`, `channelname` | Phase 1 |
| Firehose delivery | Sends processed events to Evergreen and/or Legacy Kinesis Data Firehose streams via `cnlib.firehose.Firehose` | **Rebuild Improved** | Kafka producer in `services/kafka_producer.py` using `confluent-kafka`. `acks=all`, idempotence enabled. Dead-letter topic for failures | Events delivered to Kafka topic; delivery confirmed with `acks=all`; failed deliveries routed to dead-letter topic; output JSON format identical to legacy Firehose records | Phase 1 — Kafka replaces Firehose per PRD |
| Debug Firehose delivery | Sends to debug Firehose streams (`DEBUG_EVERGREEN_FIREHOSE_NAME`, `DEBUG_LEGACY_FIREHOSE_NAME`) when `TVEVENTS_DEBUG=true` and channel is obfuscated | **Rebuild Improved** | Debug Kafka topic for obfuscated events when debug mode enabled. Single debug topic replaces dual debug Firehose streams | Debug events (obfuscated channels) delivered to debug Kafka topic when enabled; debug mode configurable via environment | Phase 1 — simplified debug routing |
| Parallel Firehose sending | `ThreadPoolExecutor` for concurrent delivery to multiple Firehose streams | **Rebuild Improved** | `confluent-kafka` producer handles async delivery natively via internal produce queue and background thread. No explicit thread pool needed | Kafka producer delivers to topic without blocking request handler; delivery callbacks confirm success/failure asynchronously | Phase 1 — native Kafka async replaces thread pool |
| RDS PostgreSQL connection | `psycopg2` with env var config (`RDS_HOST`, `RDS_DB`, `RDS_USER`, `RDS_PASS`, `RDS_PORT`). Secrets from AWS Secrets Manager | Connections pooled and reused; pool exhaustion returns error (not hang); connection timeout configured; pool closed on shutdown | Phase 1 — async pooling replaces per-query connections |
| Request logging | `before_request` Flask hook logs method, path, IP, headers for every request | **Rebuild Improved** | FastAPI middleware logs method, path, IP, and selected headers with OTEL trace correlation (`trace_id`, `span_id`). | Error handling | `TvEventsCatchallException` wraps all errors with 400 HTTP status; generic error message to client | **Rebuild Improved** | FastAPI exception handlers with typed Pydantic error responses. Distinguish 400 (validation), 422 (schema), 500 (internal). Suppress internal details in client responses; full details logged server-side | Validation errors return 400; schema errors return 422; internal errors return 500 with generic message; no stack traces or internal paths in responses | Phase 1 |
| OTEL instrumentation | Traces on all functions, custom metrics counters (DB connections ×4, cache operations ×2, firehose sends ×1, payload validation ×4). | OTEL metrics export | OTLP HTTP exporter to New Relic endpoint | **Rebuild Improved** | OTLP gRPC to OTEL Collector sidecar. ### Intentionally Dropped

| Feature | Legacy Behavior | Status | Justification |
|---|---|---|---|
| PagerDuty integration | `pygerduty` library (service ID `PSV1WEB`) for embedded application-level alerting | **Intentionally Dropped** | Alerting is an infrastructure concern, not an application concern. OTEL metrics + SRE agent external alerting replaces embedded PagerDuty. See ADR-008 |
| cnlib.log logging | Custom logger via `cnlib.log.getLogger` with file-based rotation | **Intentionally Dropped** | Replaced by Python stdlib `logging.getLogger` with OTEL structured JSON handler. cnlib logger provided no functionality beyond stdlib + file rotation, which is unnecessary in containerized environments |
| cnlib.firehose integration | `cnlib.firehose.Firehose` — Kinesis Data Firehose wrapper using boto3 | **Intentionally Dropped** | Kafka replaces Firehose per organizational strategy. Standalone `services/kafka_producer.py` replaces cnlib wrapper. See ADR-002 |
| cnlib.token_hash integration | `cnlib.token_hash.security_hash_match` — HMAC hash validation via shared library | **Intentionally Dropped** | Standalone `domain/validation.py security_hash_match()` using `hmac.compare_digest()`. cnlib dependency removed entirely. See ADR-005 |
| google-cloud-monitoring | Legacy Stackdriver dependency present in requirements | **Intentionally Dropped** | Not used by this application. Vestigial dependency from cnlib. OTEL replaces all monitoring |
| Legacy Firehose routing complexity | Env-var-driven routing to multiple Firehose streams (`SEND_LEGACY`, `SEND_EVERGREEN`, `DEBUG_*`) with conditional logic | **Intentionally Dropped** | Simplified to single Kafka topic + debug topic. Stream multiplexing is a downstream concern, not an application concern |

### Status Definitions

- **Must Rebuild** — feature must exist in the target with equivalent behavior
- **Rebuild Improved** — feature will be rebuilt with specific improvements
- **Intentionally Dropped** — feature will not be rebuilt (requires documented justification below)
- **Deferred** — feature will be rebuilt in a later phase

## Intentionally Dropped — Justifications

| Feature | Justification |
|---|---|
| PagerDuty integration (`pygerduty`) | Unmaintained library. Alerting is an infrastructure concern handled externally by OTEL metrics export + SRE agent alerting policies. The rebuilt application emits standardized telemetry; it does not page on-call engineers directly. See ADR-008 |
| cnlib.log logging | cnlib `log.getLogger` wraps stdlib `logging` with file-based rotation and custom formatting. In containerized environments, logs go to stdout/stderr and are collected by the platform. Python stdlib `logging.getLogger` with OTEL structured JSON handler provides equivalent functionality plus trace correlation |
| cnlib.firehose integration | `cnlib.firehose.Firehose` wraps `boto3` Kinesis Data Firehose `put_record`. Organization is migrating to Apache Kafka. Standalone `services/kafka_producer.py` using `confluent-kafka` replaces this with `acks=all`, idempotence, and dead-letter topic. See ADR-002 |
| cnlib.token_hash integration | `cnlib.token_hash.security_hash_match` is ~15 lines of HMAC logic. Inlined as standalone `domain/validation.py security_hash_match()` with `hmac.compare_digest()` for constant-time comparison — a security improvement. See ADR-005 |
| google-cloud-monitoring | Vestigial Stackdriver dependency. Not imported or used anywhere in the evergreen-tvevents application code. Carried in as a cnlib transitive dependency |
| Legacy Firehose routing complexity | The `SEND_LEGACY`/`SEND_EVERGREEN`/`DEBUG_*` env var matrix controlling delivery to 4+ Firehose streams is replaced by Kafka topic routing. Downstream consumers read from Kafka topics instead of S3 paths written by Firehose. Stream fan-out is a Kafka consumer concern |
