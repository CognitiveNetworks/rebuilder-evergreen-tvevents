> Each feature gets a status. The rebuild is not complete until every **Must Rebuild** and **Rebuild Improved** feature passes acceptance testing.

### Status Definitions

| Status | Meaning |
|---|---|
| **Must Rebuild** | Feature exists in legacy and must be reproduced with identical behavior in the rebuild. No behavioral change permitted. |
| **Rebuild Improved** | Feature exists in legacy but is rebuilt with a better implementation. Functional behavior is equivalent; non-functional properties (performance, maintainability, observability) are improved. |
| **New** | Feature does not exist in legacy. Added in the rebuild to address gaps identified in the legacy assessment or PRD. |
| **Deferred** | Feature is recognized as valuable but explicitly scoped out of Phase 1. Tracked here to prevent scope creep and ensure it is addressed in a future phase. |

## Intentionally Dropped — Justifications

| Feature | Justification |
|---|---|
| cnlib.firehose dependency | Replaced by standalone Kafka module. All Firehose delivery functionality is provided by the Kafka module with topic-based routing. JSON payloads are byte-for-byte identical — only the transport changes. Per PRD Goal #2: decouple from AWS-specific delivery service. |
| Flask framework | Replaced by FastAPI. Per PRD Goal #1. FastAPI provides native async support, Pydantic validation, automatic OpenAPI generation, and dependency injection — none of which Flask offers natively. |
| Gunicorn + gevent runtime | Replaced by uvicorn. ASGI-native server eliminates gevent monkey-patching. Simpler process model, better async I/O support. Template-repo-python standard runtime. |
| Manual OTEL setup (48 lines) | Replaced by OTEL auto-instrumentation via `FastAPIInstrumentor`. Per PRD Goal #5. Auto-instrumentation covers more surface area with zero boilerplate. Custom metrics are retained and ported separately. |
| Flat requirements.txt | Replaced by pip-compile workflow with hash pinning via `scripts/lock.sh`. Per PRD Goal #9. Hash pinning prevents supply-chain attacks. Transitive dependencies are fully resolved and pinned. |

**Rules applied:**
- Every function consumed from a legacy library is mapped to its target replacement
- No library is described as "removed" without showing its replacement
- All replacements are confirmed based on PRD design decisions and template-repo-python patterns
